# Инструкция по добавлению статьи в справочник

## 1. Данные автора

При отсутствии данных об участнике проекта их необходимо добавить в `data/team.json` в следующем формате:

```json
{
  "id": "ivan-petrov",
  "name": "Иван Петров",
  "email": "ipetrov@mail.ru",
  "role": "author",
  "photo": "ivan-petrov.jpg",
  "articles": ["statia-1", "statia-2"] # идентификатор статьи
}
```

### Добавление фотографии

- если вы планируете добавить свою фотографию для отображения на странице справочника, добавьте файл фотографии в папку `assets/images`;
- если вы не хотите добавлять фотографию, заполните данные следующим образом: `"photo": "",`.

### Данные автора есть в базе

Необходимо добавить только идентификатор статьи в список `articles`.

## 2. Иерархия справочника

Данный шаг **нужен** только в тех случаях, когда статья не входит не в один из разделов и/или подразделов справочника. Если же вы нашли в `handbook.json` уже есть нужный раздел и/или подраздел, данный пункт необходимо пропустить.

### Добавление нового раздела

Если нужного раздела не существует, добавьте новый объект в массив `sections`:

```json
{
  "section": "Название раздела",
  "id": "nazvanie-razdela",
  "shortDescription": "Краткое описание раздела",
  "description": [
    {
      "type": "paragraph",
      "text": "Просто текст"
    },
    {
      "type": "list",
      "items": ["элемент-1", "элемент-2"]
    }
  ],
  "subsections": []
}
```

### Добавление нового подраздела

Если раздел существует, но нет подходящего подраздела, добавьте новый объект в массив `subsections` нужного раздела:

```json
{
  "subsection": "Название подраздела",
  "articles": []
}
```

## 3. Подготовка метаданных

Добавьте информацию о статье в `data/handbook.json` в нужный подраздел:

```json
{
  "id": "statia-1",
  "title": "Название статьи 1",
  "authorID": "ivan-petrov",
  "tags": ["тег1", "тег2", "тег3"],
  "markUp": "statia-1.tsx",
  "sectionID": "grammar-forms"
}
```

### id

Идентификатор статьи должен представлять собой траслитерацию названия статьи.

**NB:** все пробелы и знаки препинания должны быть заменены на `-`.

---

### title

Название статьи должно быть дано на русском языке. Может быть использована пунктуация. Для добавления курсивного текста нужно обернуть нужный фрагмент названия в `<span class='italicText'>ВАШ ТЕКСТ</span>`

---

### authorID

Идентификатор автора должен совпадать с идентификатором в `team.json`.

---

### tags

В данное поле необходимо добавить ключевые слова статьи. Каждый элемент списка `tags` должен быть обернут в двойные кавычки.

**NB:** не использовать html-теги.

---

### markUp

В данное поле необходимо добавить название файла, в котором будет лежать разметка содержания статьи. Название файла **должно** совпадать с идентификатором статьи из поля `id`.

---

### sectionID

Каждая статья в справочнике должна лежать в одном из разделов статьи. Для того, чтобы узнать раздел и подраздел (местоположение статьи в справочнике), необходимо обратиться к руководителю проекта.

Добавьте в `sectionID` идентификатор раздела.

## 4. Добавление содержании статьи

Создайте файл в директории `components/articles/` с именем `nazvanie-statii.tsx` (оно должно совпадать с `markUp` из `handbook.json`).

Вставьте в него данный код.

```tsx
import { FC } from "react";
import { TextBlock } from "../components/ui/article-blocks/text-block/text-block";
import { Paragraph } from "../components/ui/article-blocks/paragraph/paragraph";
import { Quote } from "../components/ui/article-blocks/quote/quote";
import { References } from "../components/ui/article-blocks/references/references";
import { Question } from "../components/ui/article-blocks/question/question";
import { Link } from "../components/ui/article-blocks/link/link";

/* Заменить название компонента на название вашей статьи  */
const NazvanieStatiContent: FC = () => {
  return <>{/* Содержание статьи */}</>;
};

/* Заменить название компонента на название вашей статьи  */
export default NazvanieStatiContent;
```

Далее вам предстоит вставить содержание статьи. Для этого нужно использовать готовые компоненты и заполнить их данными.

### TextBlock - текстовый блок с заголовком

```tsx
<TextBlock title="Заголовок раздела" id="unikalyj-id">
  <Paragraph>Текст параграфа 1</Paragraph>
  <Paragraph>Текст параграфа 2</Paragraph>
</TextBlock>
```

### Paragraph - параграф текста

```tsx
<Paragraph>
  Обычный текст с <b>жирным</b> и <i>курсивным</i> форматированием.
</Paragraph>
```

### Quote - цитата с примером

```tsx
<Quote
  number="1а"
  source="Источник цитаты"
  text={
    <Paragraph>
      Текст цитаты с <b>выделением</b>.
    </Paragraph>
  }
/>
```

### Question - блок с конкретным вопросом

```tsx
<Question>
  <Paragraph>Формулировка вопроса</Paragraph>
</Question>
```

### References - список литературы

```tsx
<References
  items={[
    <Paragraph>Автор. Название работы. Год.</Paragraph>,
    <Paragraph>
      Другой автор. <Link href="https://example.com" text="Название ссылки" />
    </Paragraph>,
  ]}
/>
```

### Link - внешняя ссылка

```tsx
<Link href="https://example.com" text="Текст ссылки" />
```

### Общие правила оформления статьи

**Идентификаторы (id)**

- Должны быть уникальными в пределах статьи
- Используются для навигации по статье
- Нужно использовать латиницу и дефисы: "main-section"

**Нумерация примеров**

- Используйте последовательную нумерацию: "1", "2", "3"
- Для подпримеров используйте: "1а", "1б", "1в"

**Форматирование текста**

- \<b>...\</b> для жирного выделения
- \<i>...\</i> для курсива
- Не использовать HTML-тегов кроме указанных
